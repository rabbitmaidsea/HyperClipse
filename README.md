# HyperClipse AI Engine

> **The creator owns creativity.**
>
> **HyperClipse preserves continuity.**

HyperClipse is a lightweight runtime derived from **CONSEQUENCE AI 2 Canon**, designed to help language models preserve identity, continuity, and long-term consequences throughout AI-driven narratives.

---

## Why HyperClipse Exists

Language models are remarkably creative.

However, over time they often begin to lose continuity.

Characters forget who they are.

Relationships lose their history.

Worlds forget their own past.

Consequences disappear.

HyperClipse was created to help preserve those things.

It does not replace creativity.

It protects the consequences created by creativity.

---

## Philosophy

HyperClipse is built around a simple principle.

The creator decides **what happens**.

HyperClipse attempts to preserve **what those decisions mean**.

If trust is broken, relationships should remember.

If a city falls, history should remember.

If years pass, the world should continue moving forward.

The runtime does not decide your story.

It simply helps your story remain consistent as it grows.

---

## What HyperClipse Is

HyperClipse is a declarative runtime built from architectural invariants.

Rather than describing every possible behavior, it defines a compact set of persistent rules that guide narrative consistency.

Examples include:

* Identity persists.
* Trust is earned.
* Observe before knowing.
* Relationships evolve gradually.
* History matters.

Each invariant exists because it solved an observed engine failure during RabbitMaid research.

---

## What HyperClipse Is Not

HyperClipse intentionally does **not**:

* Write stories.
* Generate characters.
* Generate worlds.
* Replace creator intent.
* Replace platform infrastructure.
* Replace memory systems.
* Implement databases or storage.

HyperClipse is a runtime.

The creator remains the storyteller.

---

## Compatibility

HyperClipse has been validated on platforms that provide a dedicated **Scenario** (or equivalent system-level prompt) field.

Other prompt locations are expected to function, but have not yet been systematically evaluated.

Different prompt execution models may influence long-term runtime stability.

---

## Development

HyperClipse is derived from **CONSEQUENCE AI 2 Canon**.

The Canon serves as the project's reference architecture and source of truth.

HyperClipse is the executable runtime distilled from that architecture.

Future development remains bug-driven.

New invariants are introduced only when justified by observed engine failures.
